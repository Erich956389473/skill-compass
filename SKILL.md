# Skill Compass — 技能指南针

## 它是干什么的

市面上有几百个 Agent Skill，但没人告诉你**哪个适合你的场景**、**哪些应该组合在一起用**、**还缺什么**。

Skill Compass 不教你怎么用 Skill，它帮你在 Skill 海洋里找到方向。

## 怎么用

在你开始一个任务之前，告诉 Skill Compass 你想做什么：

```
我要做一个 SaaS 产品的着陆页，用 Claude Code 生成
```

Skill Compass 会返回：

```
推荐 Skill 组合：
1. hallmark → 避免AI模板味（你的场景需要高频使用）
2. fe-inspector → 检查可访问性和响应式（着陆页需要SEO友好）
3. archify → 可选，如果你需要出架构图

组合建议：
hallmark 生成时用 → fe-inspector 生成后检查
缺什么：
- 你的场景没有专门的"着陆页文案"Skill，
  建议你手动写一段品牌文案指导给Agent
```

## 推荐逻辑

Skill Compass 通过以下维度匹配合适的 Skill：

1. **任务类型匹配** — 你做的事属于哪个领域（前端/PM/数据分析/内容/基础设施）
2. **场景专精度** — 这个 Skill 在这个场景下表现如何
3. **互补性** — 多个 Skill 能否组合出更好的效果
4. **平台兼容性** — 你的 Agent 是否支持这个 Skill

## 常见任务 → Skill 组合推荐

### 场景 1：用 AI 生成前端页面

| 需求 | 推荐 Skill | 原因 |
|------|-----------|------|
| 生成时避免AI模板味 | **hallmark** | 57条审美规则，防紫色渐变/过度圆角 |
| 生成后检查质量 | **fe-inspector** | 查可访问性+响应式+交互+审美，20条规则 |
| 需要出架构图 | **archify** | 如果有系统设计图的需求 |
| 检查代码依赖 | **code-review-graph** | 大型项目才需要，小项目跳过 |

**最佳组合：** hallmark（生成时）+ fe-inspector（生成后检查）
**还缺什么：** 没有专门的 AI 生成内容文案质量检查 Skill

## Skill 详细目录

详细的 Skill 元数据在 `skills-index.json` 中，包含每个 Skill 的：
- 适用场景（bestFor）
- 不适用场景（notFor）
- 平台要求
- 优点和缺点

当用户描述的任务不在上面的常见场景中时，请参考 `skills-index.json` 进行匹配。

## 匹配算法（给 Agent 的指引）

当你收到用户的请求时，按以下步骤处理：

### 步骤 1：理解用户意图
从用户描述中提取：要做什么类型的任务？哪个领域？输出是什么？

### 步骤 2：匹配 Skill
在 `skills-index.json` 中搜索符合条件的 Skill：
- 检查 bestFor 是否包含用户的场景关键词
- 排除 notFor 中包含的场景
- 优先选择 platform 与用户 Agent 兼容的

### 步骤 3：组合建议
如果多项 Skill 互补，推荐组合使用，并说明每个 Skill 在哪个环节发挥什么作用。

### 步骤 4：识别空白
如果用户的场景没有完全匹配的 Skill，明确说出来："这个功能目前还没有专门的 Skill，但你可以..."
这既是帮助用户，也是给 Skill 开发者指明方向。

### 步骤 5：输出格式

```
推荐 Skill 组合：
1. [skill名称] → [一句话说明为什么适合]
2. [skill名称] → [一句话说明为什么适合]

组合建议：
[如何组合使用]

缺什么：
[如果用户的场景没有Skill覆盖，明确指出]
```

## 设计原则

1. **不偏袒** — 不因为是我做的 Skill 就优先推荐，根据场景客观匹配
2. **诚实** — 如果某个 Skill 不适合用户场景，直接说出来
3. **组合思维** — 单一 Skill 的局限性通过组合来弥补
4. **看见空白** — 识别没有 Skill 覆盖的场景，这是生态的机会
